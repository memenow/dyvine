Sequence diagram

Component interactions for livestream and bulk-post downloads, including the polling path clients use to observe progress.

Updated 2026-05-10

Both flows run as fire-and-forget background tasks — the API returns a 202 with an operation_id immediately and the heavy lifting happens on a tracked asyncio.Task spawned through the shared BackgroundTaskRegistry.

Operational notes

  • BackgroundTaskRegistry.drain runs in the FastAPI lifespan's shutdown branch with a 30-second budget; pending tasks are cancelled with structured logging if they overrun.
  • The sticky-cursor guard inside PostService._run_bulk_download stops pagination when the upstream max_cursor equals the input — Douyin echoes it instead of advancing — so the loop never busy-waits.
  • The R2 archival path is only taken when every R2 setting is configured; otherwise R2StorageService auto-disables and downloads stay on local disk under DOUYIN_DOWNLOAD_ROOT.
For the conditional branches taken inside _resolve_webcast_id and _run_bulk_download, see the control-flow graph.
