Dyvine Architecture

Reference diagrams for the Dyvine FastAPI service.

Last reviewed: 2026-05-29 Source layout: src/dyvine/ Entry point: main.py

Dyvine fronts the third-party f2 Douyin SDK with a FastAPI service that delivers asynchronous content downloads, persistent operation tracking, and optional Cloudflare R2 archival. These diagrams describe the runtime architecture as it is implemented in main.

Layered architecture at a glance

  • Presentation — FastAPI routers under src/dyvine/routers. All router endpoints sit behind an X-API-Key dependency and the handle_errors decorator.
  • Service — UserService, PostService, LivestreamService, R2StorageService, and LifecycleManager in src/dyvine/services.
  • Persistence — OperationStore (SQLite + WAL) at src/dyvine/core/operations.py; per-thread reader connections, single writer guarded by a lock, dispatched onto a dedicated sqlite_executor.
  • Infrastructure — ServiceContainer wires four named thread-pool executors (R2, R2-head, SQLite, audit), a BackgroundTaskRegistry, and the f2 DouyinHandler.
  • Observability — Prometheus metrics (/metrics), structured JSON logging, request correlation IDs, split /livez//readyz//startupz probes.

Operational invariants

  • Every long-running download returns immediately as a persisted OperationRecord and continues on a tracked background task.
  • The lifespan drains the BackgroundTaskRegistry before reaping executor pools, so an active R2 upload always finishes (or is cancelled with structured logging) before the process exits.
  • User-supplied filesystem paths are jailed inside DOUYIN_DOWNLOAD_ROOT; symlink redirection, traversal segments, and absolute prefixes are rejected pre- and post-mkdir.
  • The composite Settings validator refuses to boot in non-debug mode when SECURITY_SECRET_KEY or SECURITY_API_KEY still match the placeholder sentinel.
  • OperationStore.mark_incomplete_operations_failed sweeps stale pending/running rows to failed on startup so a crashed deployment never leaves orphaned in-progress rows behind.
