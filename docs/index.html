Overview

Dyvine fronts the third-party f2 Douyin SDK with a REST API that schedules long-running work, persists operation state, and exposes operational health probes. It supports downloads for videos, image galleries, livestreams, and user content.

  • Async, fire-and-forget downloads return an operation_id immediately and continue on tracked background tasks.
  • Persistent operation state is stored in a WAL-mode SQLite database.
  • Router endpoints are protected by X-API-Key authentication by default.
  • User-supplied output paths are resolved inside DOUYIN_DOWNLOAD_ROOT and rejected when they escape the configured root.
  • Dedicated thread pools isolate R2 calls, R2 metadata fan-out, SQLite work, and audit-log writes.
  • Operational endpoints include /livez, /readyz, /startupz, /health, and /metrics.

Quick Start

Prerequisites

  • uv or Python 3.12+
  • Git
  • A valid Douyin session cookie
  • Optional Cloudflare R2 credentials and bucket for archival uploads

Install

git clone https://github.com/memenow/dyvine.git
cd dyvine
uv sync --all-extras

Run

uv run uvicorn src.dyvine.main:app --reload

For a production-style process, run:

uv run uvicorn src.dyvine.main:app --host 0.0.0.0 --port 8000 \
    --timeout-graceful-shutdown 25

Configuration

Copy .env.example to .env and populate required fields.

cp .env.example .env

Minimum required settings for a non-debug deployment:

API_DEBUG=false
SECURITY_SECRET_KEY=<48+ bytes of entropy>
SECURITY_API_KEY=<48+ bytes of entropy>
DOUYIN_COOKIE=<browser session cookie>

Generate strong local secrets with:

python -c "import secrets; print(secrets.token_urlsafe(48))"

Optional R2 archival requires R2_ACCOUNT_ID, R2_ACCESS_KEY_ID, R2_SECRET_ACCESS_KEY, R2_BUCKET_NAME, and R2_ENDPOINT. The storage service auto-disables when any R2 field is empty. The readiness probe fails until all R2 fields and the Douyin cookie are set.

API Reference

Router endpoints sit under the configurable API_PREFIX, which defaults to /api/v1.

Authentication

When SECURITY_REQUIRE_API_KEY=true, every router request must include the configured key in the X-API-Key header. Mismatches return 401 Unauthorized. 

X-API-Key: <SECURITY_API_KEY>

System Endpoints

GET /                       # Service metadata and feature list
GET /livez                  # Process liveness signal
GET /readyz                 # Dependency-aware readiness
GET /startupz               # Lifespan startup completion
GET /health                 # Aggregated runtime metrics

Posts

GET  /api/v1/posts/{post_id}
GET  /api/v1/posts/users/{user_id}/posts
POST /api/v1/posts/users/{user_id}/posts:download
GET  /api/v1/posts/operations/{operation_id}

The list endpoint accepts an opaque page_token query parameter. Echo the response's next_page_token back unchanged on the follow-up call.

Livestreams

POST /api/v1/livestreams/users/{user_id}/stream:download
POST /api/v1/livestreams/stream:download
GET  /api/v1/livestreams/operations/{operation_id}

The URL endpoint validates the host against an allowlist of *.douyin.com domains to prevent SSRF.

Operations

Persistent Operation Store

src/dyvine/core/operations.py owns a SQLite database at data/douyin/state/operations.db by default. Override it with API_OPERATION_DB_PATH.

  • WAL journaling allows concurrent readers and a single writer.
  • Per-thread reader connections allow parallel operation polling.
  • A shared writer connection is guarded by a lock.
  • Startup sweeps pending and running rows to failed.
  • SQLite calls run on a dedicated dyvine-sqlite executor.

Background Task Lifecycle

Long-running downloads are scheduled through BackgroundTaskRegistry. During shutdown, the FastAPI lifespan drains tracked tasks before reaping executor pools.

Deployment

Docker

docker build -t dyvine:latest .

docker run -d \
  --name dyvine \
  -p 8000:8000 \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/logs:/app/logs \
  --env-file .env \
  --restart unless-stopped \
  dyvine:latest
  • The image is multi-stage and runs as non-root user 1000.
  • The runtime layer uses python -m uvicorn with --timeout-graceful-shutdown 25.
  • The build removes the f2-injected black binary from the production layer.

Kubernetes

Kustomize manifests live under k8s/ with a base and a production overlay.

kubectl apply -k k8s/base
kubectl apply -k k8s/overlays/production
Scaling note: The default operation store is pod-local SQLite backed by a ReadWriteOnce PVC and a single writer. Replace it with a shared backend before scaling beyond one replica.

Security Notes

  • API key auth is enabled by default.
  • Non-debug settings reject placeholder production secrets.
  • Livestream URL schema validation only accepts douyin.com family hosts.
  • Output paths are checked before and after directory creation, including symlink segment detection.
  • /health and /livez are informational; use /readyz for dependency-aware gating.

Contributor workflow: create a feature branch from main, run make format && make lint && make test, and open a pull request.
